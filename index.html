Hollywood

A distributed test framework
supporting multi-device tests.

Parallel scalability of tests across a finite number of resources.

Hollywood Gives You:

  • Job distribution
  • Device resource scheduling
  • Results / reporting

Hollywood Does NOT Give You:

  • A test runner
  • Device manager/controller

Job Submission

Submit jobs - or batches of jobs - through a well defined API.

  • Continuous integration
  • Manually

Batch

  • Title
  • List of Jobs

Job

  • Title
  • Config (metadata)*
  • Device under test
*Hollywood passes the config around, but isn't concerned with the contents.

Device

  • ID
  • Tags
  • Config*
  • List of Supporters (additional devices)
*Hollywood passes the config around, but isn't concerned with the contents.

Device Agent

A device agent sits outside of Hollywood and commuicates the state of the device(s) it represents.

Needs to be able to spawn a test runner for devices it represents.

Should be written by the team most familiar with the underlying device.

Communication to Hollywood

  • Device Agent is connected.
  • Specific device is potentially available.
  • Specific device is no longer available.

Communication from Hollywood

  • Reserve a specific device
  • Execute a job (specific device under test)
  • Release a specific device
The device agent should notify Hollywood of the success/failure of each of these actions.

Test Runner

The test runner sits outside of Hollywood, is spawned by the device agent with a job to run, along with the devices that have been reserved.

Uses the job/device config and run the job.

Could be as simple as a standard Jasmine Test Runner, or as complex as the Chromecast E2E Test Runner.

Should be written by the team testing underlying device.

Messaging Service

Provides a socket for communication, so that tests can be run across device agents.

Using the messaging service is optional, and typically only required by test runners controlling multiple devices.

Queue

Internal to Hollywood, the queue will find pending jobs which can be executed by the devices available.

Ensures the fastest possible execution of all pending jobs with a fair distribution*.

*The appropriate algorithm to achieve this may take some time to realize.

Matching

Devices are matched with jobs through tags.

  • Device tags are provided by the device agent
  • Job tags are provided when submitting the job

Results / Reporting

Hollywood provides an interface for publishing job results.

This data is then aggregated into simplified results, allowing grouping up (see job results, batch results, etc).

Access to original raw data is always available

Summaries

With the concept of batching, Hollywood can report the success/failure of a batch and summarize the jobs

  • Results page
  • Send an email
  • Etc
Results are mapped to a generic format, and therefore each project doesn't need to implement their own email reporter.
